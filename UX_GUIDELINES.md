# 任务列表操作按钮 UX 设计指南

## 📱 何时使用下拉更多按钮

### ✅ **推荐使用下拉菜单的场景**

#### 1. **紧凑模式 (Compact Mode)**
- **使用位置**：项目详情页的任务列表
- **原因**：
  - 空间极度有限，需要最大化信息密度
  - 用户主要目的是快速浏览任务状态
  - 编辑/删除操作相对低频
- **实现**：`mode="compact"`

#### 2. **看板模式 (Kanban Mode)**  
- **使用位置**：任务看板视图
- **原因**：
  - 卡片空间有限，主要交互是拖拽
  - 状态切换是主要操作，编辑删除是次要操作
  - 保持卡片视觉简洁性
- **实现**：`mode="kanban"`

#### 3. **移动端设备**
- **使用位置**：任何模式在移动设备上
- **原因**：
  - 屏幕空间宝贵
  - 触摸操作需要更大的点击区域
  - 减少界面拥挤感
- **实现**：自动检测屏幕宽度 < 640px

#### 4. **强制场景**
- **使用位置**：特殊需求时
- **实现**：`force-dropdown="true"`

### ❌ **不推荐使用下拉菜单的场景**

#### 1. **桌面端列表模式 (List Mode on Desktop)**
- **原因**：
  - ✅ 屏幕空间充足
  - ✅ 编辑/删除是相对高频操作
  - ✅ 减少点击步骤（直接点击 vs 展开-点击）
  - ✅ 提高操作效率
  - ✅ 按钮始终可见，降低认知负担

## 🎯 设计原则

### 操作频率 vs 空间效率
```
高频率操作 + 充足空间 → 直接显示按钮
低频率操作 + 有限空间 → 下拉菜单
```

### 用户任务优先级
```
主要任务（浏览、状态切换）→ 优化其显示区域
次要任务（编辑、删除）→ 可以收纳到下拉菜单
```

### 设备特性适配
```
桌面端：优先考虑操作效率
移动端：优先考虑空间利用
```

## 📊 当前实现策略

| 模式/设备 | 按钮显示方式 | 理由 |
|-----------|-------------|------|
| Desktop + List | 直接显示 | 高频操作 + 充足空间 |
| Mobile + List | 下拉菜单 | 空间限制 |
| Any + Kanban | 下拉菜单 | 主要操作是拖拽 |
| Any + Compact | 下拉菜单 | 极度空间限制 |
| Any + 强制模式 | 下拉菜单 | 特殊需求 |

## 🔄 响应式行为

组件会自动根据设备类型调整：
- **桌面端** (≥640px)：List模式显示直接按钮
- **移动端** (<640px)：所有模式使用下拉菜单
- **自动检测**：无需手动配置

## 🎨 视觉设计

### 下拉菜单激活状态
- 背景变暗，提供明确的视觉反馈
- 平滑的淡入动画
- 玻璃态效果保持设计一致性

### 直接按钮
- 保持原有设计风格
- 悬停效果提供交互反馈
- 危险操作（删除）使用红色警告色

## 🚀 未来优化建议

1. **用户偏好记忆**
   - 记住用户展开/收起的偏好
   - 提供设置选项让用户自定义

2. **智能默认状态**
   - 根据用户操作频率动态调整
   - 常用操作可以"提升"到直接显示

3. **快捷键支持**
   - 桌面端支持键盘快捷键
   - 提高高级用户操作效率

4. **批量操作**
   - 长按或勾选多个任务
   - 批量编辑/删除功能
